# Sociology LLM Knowledge Base — Agent Instructions

## Project Identity

This is a **Karpathy-style LLM Knowledge Base** for a full Sociology degree.
The LLM owns all wiki content. Humans rarely edit the wiki directly.
The cycle is: **raw data → LLM compile → wiki → query/Q&A → file answers back → enhance wiki**.

**Domain**: Sociology only. Every semester's courses feed into one cumulative wiki.
**Primary IDE**: Obsidian (the `data/` directory is the vault).
**Canonical layer**: Markdown files under `data/wiki/`.
**Raw evidence layer**: PDFs, text, web clips under `data/raw/`.
**Derived layer**: Graph, search index, slides, visualizations — all regenerated, never hand-edited.

## Architecture Rules

1. **Markdown is source of truth.** Notion, Streamlit, and any other UI are downstream consumers.
2. **Raw files are immutable evidence.** Never modify files in `data/raw/`.
3. **Wiki notes use `[[wikilinks]]`.** Concept references use `[[concept-slug|Display Name]]`, author references use `[[author-slug|Author Name]]`, source references use `[[source-slug|Title]]`. This is Obsidian-native.
4. **Derived artifacts are disposable.** `data/graph/`, concept/author/course index pages, `INDEX.md`, and `GRAPH.md` can be rebuilt from source notes at any time via `rebuild_indexes()`.
5. **Safe writes via PR workflow.** Destructive edits go through branches, never direct mutation of compiled notes.
6. **No vendor lock-in.** LLM provider is configurable (OpenAI, Anthropic, or fallback mode with no API).
7. **Queries add up.** Q&A answers can be "filed back" into the wiki as `note_type: research` articles, enriching the knowledge base for future queries.
8. **Semester accumulation.** Concepts and authors accumulate across semesters. When a concept already exists from a previous semester, merge into the existing page — never duplicate.

## Directory Structure

```
data/                              ← Obsidian vault root
  .obsidian/                       ← Obsidian config (app.json, graph.json, community-plugins.json)
  raw/<semester>/<course>/         ← immutable source files + .meta.json sidecars
  wiki/
    sources/<semester>/<course>/*.md  ← compiled source notes (canonical)
    concepts/*.md                  ← derived concept aggregation pages
    authors/*.md                   ← derived author aggregation pages
    courses/*.md                   ← derived course aggregation pages
    research/*.md                  ← filed Q&A answers and explorations (note_type: research)
    slides/*.md                    ← Marp-format slide decks (marp: true frontmatter)
    assets/                        ← images downloaded from web articles
    assets/viz/                    ← generated matplotlib PNG/SVG visualizations
    INDEX.md                       ← auto-maintained master index (cross_semester_index())
    GRAPH.md                       ← mermaid diagram of degree knowledge map
  qa/
    answered_questions/*.md        ← Q&A artifacts with citations
    open_questions/*.md            ← extracted open questions from source docs
  graph/
    atlas_graph.json               ← generated node/edge graph (semester, course, source, concept, author, research nodes)
    search_index.json              ← precomputed TF-IDF search index
```

## Code Layout

All Python code lives under `sociology_kb_starter/`:

### Core pipeline
| Module | Responsibility |
|--------|---------------|
| `kb_core/models.py` | Pydantic schemas: `DocumentStatus`, `ExtractionResult`, `RawDocumentMetadata`, `CompiledSourcePayload`, `CompilationResult` |
| `kb_core/config.py` | `Settings` dataclass from `.env` — all path properties (`raw_dir`, `wiki_dir`, `sources_dir`, `concepts_dir`, `authors_dir`, `courses_dir`, `research_dir`, `slides_dir`, `assets_dir`, `viz_dir`, `graph_dir`, `qa_dir`) |
| `kb_core/storage.py` | Raw file persistence, `.meta.json` sidecars, status transitions, `write_note()` |
| `kb_core/extraction.py` | PDF/TXT/MD text extraction with per-page diagnostics |
| `kb_core/compiler.py` | LLM compilation pipeline + fallback, `[[wikilink]]` generation, `rebuild_indexes()` |
| `kb_core/llm.py` | `LLMClient` — OpenAI/Anthropic abstraction with `.available()` and `.complete()` |
| `kb_core/utils.py` | `slugify()`, `wikilink()`, `wikilinks_list()`, `extract_wikilinks()`, markdown I/O, JSON I/O |

### Knowledge operations
| Module | Responsibility |
|--------|---------------|
| `kb_core/qa.py` | TF-IDF retrieval ranking, `answer_question()`, `file_answer_to_wiki()` |
| `kb_core/search_engine.py` | `SearchEngine` class with filters + snippets. CLI: `python -m kb_core.search_engine` |
| `kb_core/graph_index.py` | Derives nodes/edges from notes (semester, course, source, concept, author, research). Writes `atlas_graph.json` + `GRAPH.md` (mermaid) |
| `kb_core/lint.py` | `run_lint_checks()` (broken wikilinks, orphans, missing frontmatter, duplicates), `suggest_new_articles()` |
| `kb_core/semesters.py` | `init_semester()`, `semester_stats()`, `cross_semester_index()` → `INDEX.md` |

### Output generation
| Module | Responsibility |
|--------|---------------|
| `kb_core/slides.py` | `generate_slides()`, `slides_from_note()` → Marp markdown in `wiki/slides/` |
| `kb_core/viz.py` | `generate_concept_map()`, `generate_timeline()`, `generate_statistics()` → PNG in `wiki/assets/viz/` |
| `kb_core/images.py` | `download_article_images()` — download remote images, rewrite markdown paths to local |
| `kb_core/ingest.py` | `scan_pending()`, `compile_batch()`, `ingest_directory()`, `ingest_and_compile()`. CLI: `python -m kb_core.ingest` |

### Integration layer
| Module | Responsibility |
|--------|---------------|
| `kb_core/services.py` | Reusable operations consumed by Streamlit + MCP: search, read, list, graph neighbors, provenance, file_answer, slides, viz, semester stats, suggest articles |
| `kb_core/mcp_tools.py` | MCP tool wrappers with Pydantic input models |
| `kb_core/notion_sync.py` | Optional one-way push to Notion (downstream only) |
| `mcp_server.py` | FastMCP server — 15 tools (10 read-only + 5 write) |
| `app.py` | Streamlit dashboard (secondary UI — Obsidian is primary) |

## MCP Tools (15 total)

### Read-only (10)
- `kb_search_notes` — ranked retrieval with filters
- `kb_read_note` — full note content + frontmatter
- `kb_list_courses` / `kb_list_authors` / `kb_list_concepts` — entity listings
- `kb_get_graph_neighbors` — BFS traversal from any node
- `kb_get_recent_sources` — newest compiled notes
- `kb_get_source_provenance` — raw file → compiled note chain
- `kb_propose_note_edit` — safe edit proposal
- `kb_get_semester_stats` — counts/coverage per semester
- `kb_suggest_articles` — cross-course article suggestions

### Write (5)
- `kb_create_pr_for_note_changes` — branch + commit for PR review
- `kb_file_answer` — file Q&A answer back as research note
- `kb_generate_slides` — Marp slide deck from wiki context
- `kb_generate_viz` — concept map / timeline / stats chart

## Coding Conventions

- **Python 3.11+**, use `from __future__ import annotations`.
- **Pydantic v2** for all data validation.
- **Type hints** on all function signatures.
- **No global mutable state** except `SETTINGS` singleton in `config.py`.
- Frontmatter: `yaml.safe_dump` / `yaml.safe_load`.
- All file I/O through `kb_core/utils.py` helpers (UTF-8 encoding).
- Paths: `pathlib.Path`, relative to `SETTINGS.kb_root` when stored in frontmatter/JSON.
- Slugs: `slugify()` from `utils.py` for all IDs and filenames.
- Tests: `pytest` with the `kb_tmp` fixture for isolated filesystem. Run from `sociology_kb_starter/`.

## Testing

```bash
cd sociology_kb_starter
python -m pytest tests/ -v
```

31 tests across 13 files covering: extraction, compilation, retrieval, filing back, ingest, search engine, semesters, slides, viz, lint, graph, storage, and MCP service integration.

## When Modifying This Codebase

1. Run `python -m pytest tests/ -v` from `sociology_kb_starter/` after every change.
2. New modules get a corresponding `tests/test_<module>.py`.
3. New MCP tools: add to `mcp_tools.py` → `services.py` → `mcp_server.py`.
4. New note_type: update `models.py`, `services._note_type_dirs()`, `lint.py`, and `graph_index.py`.
5. New directories: add to `config.py` (as `@property`), `storage.py` (`REQUIRED_DIRS`).
6. Wikilinks in body text: always use `wikilink(slug, display)` from `utils.py`, never raw markdown links.
7. After modifying compilation output format, run `rebuild_indexes()` to regenerate derived pages.

## CLIs

```bash
# Bulk ingest a directory of PDFs/TXT/MD
python -m kb_core.ingest path/to/docs/ --semester 2026-S1 --course teoria-sociologica

# Compile all pending documents  
python -m kb_core.ingest --compile-pending

# Search the wiki
python -m kb_core.search_engine "anomie durkheim" --top-k 10 --format json
python -m kb_core.search_engine "solidarity" --course teoria --format text

# Build search index
python -m kb_core.search_engine "" --build-index

# Start MCP server
python mcp_server.py

# Start Streamlit dashboard
streamlit run app.py
```
